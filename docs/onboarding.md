# Onboarding Guide

## For New Team Members

### Day 1
1. Login to Backstage (GitHub or Guest)
2. Find your business system in the catalog
3. Browse existing Genies, Data Products, Dashboards
4. Read Platform Overview documentation

### Week 1
- Understand your Dev and Prod workspaces
- Try querying a Genie in Dev workspace
- Review Data Product SLAs
- Identify what is missing from the catalog

### Week 2
- Register a new component using self-service templates
- Contribute documentation
- Review data products owned by your team
- Connect with other business teams via Backstage

## For Platform Admins

### Registering New Components

1. **Genie:** Create > Register New Genie Agent
2. **Data Product:** Create > Register New Data Product
3. **Dashboard:** Create > Register New Dashboard

Each template asks for:
- Component name and description
- Business system (which of the 8 businesses)
- Environment (Dev or Prod)

## Quick Reference

| Action | Where |
|--------|-------|
| Browse all assets | Catalog > Components |
| Filter by business | Tags filter (e.g. port, agel) |
| Filter by type | Type filter (genie, data-product, dashboard) |
| Filter by env | Tags filter (dev, prod) |
| Register new asset | Create > Choose template |
| View documentation | Click any System > Docs tab |
| See team ownership | Click any Group |
